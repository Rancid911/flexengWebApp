# Практические правила Git

## Рабочая директория

- Фактический git-корень приложения — `dashboard-next-next16`.
- Все `git status`, `git diff`, commit и push команды выполняй из этого git-корня или через `git -C <path-to-dashboard-next-next16>`.
- Не используй внешний `dashboard-next` как git-репозиторий приложения.

## Безопасность изменений

- Перед работой проверь `git status`; считай неизвестные изменения пользовательскими и не откатывай их.
- Не используй destructive команды, включая `git reset --hard`, `git clean` и `git checkout --`, без явного запроса пользователя.
- Не включай unrelated изменения в commit и не переписывай историю без явного запроса.
- Не коммить секреты, `.env*`, credentials, generated build artifacts, caches, coverage, временные файлы и `.DS_Store`.

## Перед завершением

- Выполни `git status --short` и `git diff --name-only`.
- Проверь, что изменены только файлы в scope задачи.
- Перечисли пользователю созданные/изменённые файлы, выполненные проверки и оставшиеся проблемы.
